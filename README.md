## 🎯 Концепция
`Freenet.Proxy-O-Holic` — это маниакально эффективный, легковесный асинхронный HTTP/HTTPS прокси-сервер на базе **.NET 10**. Он решает проблему изоляции мобильных устройств (iOS/Android) от мощных десктопных утилит обхода DPI (таких как `zapret` / `winws.exe`).

Программа перехватывает сырой сокетный трафик со смартфона в локальной сети и перенаправляет его через сетевой стек вашего ПК. В этот момент запущенный на компьютере `winws.exe` незаметно модифицирует пакеты (применяет desync/split), возвращая жизнь заблокированным сервисам (**YouTube**, **Google Авторизация** и др.) прямо внутри официальных мобильных приложений и браузеров (Safari/Chrome). ✅

## 📊 Топология трафика
```text
[ Мобильное устройство ] (Настройка Wi-Fi Прокси: 8118)
          │
          ▼ (Сырой поток HTTP/HTTPS)
[ ПК: Freenet.Proxy-O-Holic ] ──► [ winws.exe (zapret) ]
                                          │
                                          ▼ (Модифицированные пакеты)
                                    [ Интернет ]
```

## 🛠 Ключевые фичи
* **Автоматический сканер интерфейсов** 🔍: При старте приложение само шуршит по сетевым картам ПК, отсекает виртуальный мусор и выводит готовые локальные IPv4-адреса. Больше никакой рутины с `ipconfig`.
* **Параллельный туннелинг** 🔄: Каждое подключение со смартфона изолируется в отдельный фоновый бэкграунд-таск, не блокируя основной поток сервера.
* **100% Покрытие тестами (Hardcore Coverage)** 🧪: Весь код проекта, включая логику парсинга пакетов, сетевые обрывы и даже системную точку входа `Program.cs`, полностью покрыт модульными тестами на xUnit и FluentAssertions.
* **Контекстное логирование сессий** 📊: Встроенный `ILogger` маркирует каждый запрос уникальным тегом `[#Id]`. В консоли четко видно, какой домен запрашивает мобилка (например, `CONNECT -> googlevideo.com:443`).

## 🚀 Быстрый старт

### Требования
* **.NET 10 SDK** 🏗️
* Настроенный и запущенный `zapret` (`winws.exe`) на вашем компьютере.

### Сборка и запуск сервера
Откройте терминал в корневой папке репозитория и выполните:
```bash
dotnet run --project src/Freenet.Proxy-O-Holic.csproj -c Release
```

При запуске приложение выведет подсказку для настройки телефона:
```text

      Freenet.Proxy-O-Holic успешно инициализирован!
info: AiWarevo.Freenet.ProxyOHolic.ProxyServer
      --> Прокси слушает порт: 8118
info: AiWarevo.Freenet.ProxyOHolic.ProxyServer
      --> Доступные IP-адреса этого ПК для настройки Wi-Fi на телефоне:
info: AiWarevo.Freenet.ProxyOHolic.ProxyServer
      [Беспроводная сеть]: 192.168.1.45
info: AiWarevo.Freenet.ProxyOHolic.ProxyServer
      Ожидание трафика от зависимых устройств...
```

### Запуск юнит-тестов
Чтобы убедиться в стабильности сборки, выполните команду тестирования:
```bash
dotnet test
```

### Настройка смартфона
1. Подключите телефон к тому же Wi-Fi роутеру, что и компьютер.
2. Зайдите в свойства вашей Wi-Fi сети на телефоне и найдите пункт **«Прокси»**.
3. Переключите его в режим **«Вручную»**.
4. Заполните поля данными из консоли приложения:
   * **Сервер / Имя хоста**: IP-адрес вашего ПК (например, `192.168.1.45`).
   * **Порт**: `8118`.
5. Сохраните настройки. Теперь весь заблокированный трафик мобилки оживет.

---
⚠️ **Если на телефоне пропал интернет**: Брандмауэр Windows блокирует порт `8118`. Откройте командную строку на ПК от Администратора и вставьте команду для открытия порта:
```bash
netsh advfirewall firewall add rule name="FreenetProxyOHolic" dir=in action=allow protocol=TCP localport=8118
```
